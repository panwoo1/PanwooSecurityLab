정보보안기사 2022년 06월 25일 기출문제 중 오답과 헷갈린 문항을 과목별로 정리했다.  
이미지 지문이 필요한 문항은 원문 지문을 다시 확인해야 하므로, 여기서는 핵심 개념 위주로 남긴다.

## 1과목. 시스템 보안

### 3. 안드로이드 ADB 접속 후 쓰기 가능한 디렉터리는?

**정답: ③ `/data/local/tmp/`**

ADB shell에서 일반적으로 임시 파일을 올리거나 테스트할 때 자주 쓰는 위치는 `/data/local/tmp/`이다. `/system/`은 시스템 영역이라 일반적으로 읽기 전용이고, `/data/app`은 설치된 앱 APK가 위치하는 영역으로 임의 쓰기 위치로 보기 어렵다.

### 6. 컴퓨터 시스템 하드닝 활동으로 틀린 것은?

**정답: ② 전체 시스템 백업**

하드닝은 시스템을 더 견고하게 만드는 보안 강화 작업이다. 불필요한 소프트웨어 제거, 감사 및 로깅 활성화, 보안 업데이트 적용은 하드닝에 해당한다. 백업은 사고 이후 복구를 위한 대비 활동에 가깝고, 직접적인 하드닝 활동으로 보기는 어렵다.

### 7. 기억 장치의 메모리 반입 정책

**정답: 이미지 지문 재확인 필요**

메모리 공간 할당 방식은 다음처럼 구분한다.

- 최초 적합: 처음 발견한 충분한 공간에 할당
- 최상 적합: 요청 크기를 수용하는 공간 중 가장 작은 공간에 할당
- 최악 적합: 요청 크기를 수용하는 공간 중 가장 큰 공간에 할당
- 다음 적합: 직전 탐색 위치 이후부터 다시 탐색

문제의 이미지 지문이 어떤 방식을 설명하는지에 따라 정답이 달라지므로 원문 이미지를 다시 확인한다.

### 9. SUID 프로그램과 심볼릭 링크 레이스 컨디션 공격

**정답: 이미지 지문 재확인 필요**

핵심은 TOCTOU(Time Of Check To Time Of Use)이다. SUID 프로그램이 임시 파일을 검사한 뒤 실제로 사용하는 사이의 짧은 간격에, 공격자가 심볼릭 링크를 바꿔치기하면 권한 상승이나 임의 파일 변경이 가능해질 수 있다. 번호형 단계는 이미지의 흐름을 다시 보고 “검사 후 사용 전” 구간을 고르면 된다.

### 10. Snort Rule에서 10~11번째 2바이트가 `0xFFFF`인지 검사하는 키워드

**정답: ④ `content`, `offset`, `depth`**

Snort에서 검사할 패턴은 `content`로 지정한다. 검색 시작 위치는 `offset`, 검사할 바이트 범위는 `depth`로 제한한다. 10~11번째 2바이트를 검사하는 규칙이라면 `content:"|FFFF|"`, `offset`, `depth:2` 구조로 이해하면 된다.

### 11. 매크로 바이러스에 대한 설명으로 틀린 것은?

**정답: ④ EXE 형태의 자동화된 기능을 포함한다.**

매크로 바이러스는 Word, Excel 같은 문서 파일의 매크로 기능을 악용한다. 문서 기능을 이용해 실행되고 이메일 첨부 등을 통해 퍼지는 경우가 많다. EXE 실행 파일 형태의 자동화 기능은 일반 실행형 악성코드에 더 가깝다.

### 18. Windows 시스템 인증 구성 요소

**정답: 이미지 지문 및 공식 정답 재확인 필요**

구성 요소별 역할을 구분한다.

- LSA: 로컬 보안 정책, 인증, 토큰 생성, 감사 정책 처리
- SAM: 로컬 사용자 계정과 암호 해시 DB
- SRM: 객체 접근 권한 검사, 보안 참조 모니터

지문이 “인증 처리와 보안 정책”을 묻는다면 LSA, “사용자 계정 DB”를 묻는다면 SAM, “접근 검사”를 묻는다면 SRM이다.

### 20. 인증의 방법이 아닌 것은?

**정답: ② Somewhere You Know**

대표적인 인증 요소는 다음과 같다.

- Something You Know: 비밀번호, PIN
- Something You Have: 토큰, 스마트카드, OTP
- Something You Are: 지문, 홍채, 얼굴 등 생체정보
- Something You Do: 서명, 음성, 키 입력 패턴

위치 기반 요소를 말하려면 보통 `Somewhere You Are`에 가깝다. `Somewhere You Know`는 일반적인 인증 요소 표현이 아니다.

## 2과목. 네트워크 보안

### 22. UDP Flooding 대응 방안으로 틀린 것은?

**정답: ② 도착지 IP별 임계치 기반 차단**

UDP Flooding 대응에서 임계치 기반 차단은 보통 송신지 IP, 패킷량, 트래픽 특성 등을 기준으로 적용한다. “도착지 IP별 임계치”만으로 차단하면 공격 대상 서비스 자체를 차단하는 형태가 될 수 있어 대응 방식으로 부적절하다.

### 25. 네트워크 장비 상태 확인 명령어

**정답: ② `show process`, `show controllers`, `show memory`**

`show process`는 프로세스와 CPU 사용 상태, `show controllers`는 인터페이스 컨트롤러 상태, `show memory`는 메모리 상태 확인에 사용한다. `show flash`는 플래시 저장장치 확인 명령이다.

### 38. RFID 보안 기술에서 암호 기술을 사용하는 보호대책

**정답: ③ XOR 기반 원타임 패드 기법**

Kill 명령어, 블로커 태그, Sleep/Wake 방식은 물리적 통제나 접근제어에 가깝다. XOR 기반 원타임 패드는 태그와 리더 사이의 값을 매번 다르게 처리하는 암호 기술 기반 보호대책이다.

### 39. Nmap의 운영체제 판단 기법

**정답: ④ TCP/IP 프로토콜 표준이 명시하지 않은 패킷 처리 기능의 운영체제별 구현**

Nmap OS Detection은 운영체제별 TCP/IP 스택 구현 차이를 이용하는 TCP/IP Fingerprinting 기법이다. 단순히 열린 포트나 배너 문자열만 보는 것이 아니라, 특수 패킷에 대한 응답 차이를 분석한다.

## 3과목. 어플리케이션 보안

### 49. DDoS 공격 형태 중 자원 소진 공격이 아닌 것은?

**정답: ① ICMP Flooding**

ICMP Flooding은 주로 대역폭 소진 공격으로 분류한다. SYN Flooding, ACK Flooding, DNS Query Flooding은 서버나 네트워크 장비의 처리 자원을 소모시키는 자원 소진 공격으로 볼 수 있다.

### 52. MS SQL 서버 인증 모드 중 성격이 다른 하나

**정답: ④ 트러스트되지 않은 연결(SQL 연결)을 사용한다.**

Windows 인증 모드는 Windows 계정과 SID 기반 인증을 사용한다. 반면 SQL Server 인증은 SQL Server 자체 계정과 암호를 사용하며, Windows 신뢰 연결과 다른 방식이다. 따라서 ④는 혼합 인증 또는 SQL Server 인증 성격이다.

### 56. 버퍼오버플로우 보안 대책이 아닌 것은?

**정답: ④ 포맷 스트링 검사**

버퍼오버플로우 대응에는 커널 패치, 경계 검사 컴파일러, 스택 실행 방지(NX/DEP) 등이 있다. 포맷 스트링 검사는 `printf(user_input)` 같은 포맷 스트링 취약점 대응에 가깝다.

## 4과목. 정보 보안 일반

### 63. MDC에 포함되지 않는 해시 관련 알고리즘

**정답: ④ H-MAC**

MDC는 비밀키 없이 메시지 변경 여부를 탐지하는 해시 함수 계열이다. MD, SHA, LSH는 MDC 계열로 볼 수 있지만, HMAC은 비밀키를 사용하는 MAC 계열이므로 MDC에 포함되지 않는다.

### 64. OCSP의 서비스가 아닌 것은?

**정답: ③ CRL**

OCSP는 인증서 상태를 온라인으로 실시간 확인하는 방식이다. CRL은 인증서 폐지 목록을 배포해 확인하는 방식으로, OCSP 서비스 자체가 아니다.

### 65. 접근통제정책 구성요소 설명으로 틀린 것은?

**정답: ③ 행위: 객체가 행하는 논리적 접근통제이다.**

접근통제에서 행위는 주체가 객체에 대해 수행하는 읽기, 쓰기, 실행 같은 접근 동작이다. 객체가 행위를 수행하는 것이 아니라 주체가 객체에 접근한다는 구조로 이해해야 한다.

### 68. 전자서명을 적용한 예에 해당되지 않는 것은?

**정답: ④ Kerberos Protocol**

Code Signing, X.509 인증서, SSL/TLS는 전자서명 또는 인증서 기반 검증과 관련이 있다. Kerberos는 대칭키와 티켓 기반 인증 프로토콜로, 전자서명 적용 예로 보기 어렵다.

### 70. RSA를 이용한 키 공유 설명으로 틀린 것은?

**정답: ④ A는 자신의 공개키로 공유 비밀키를 추출한다.**

RSA에서 암호화는 공개키로 하고 복호화는 개인키로 한다. B가 A의 공개키로 공유 비밀키를 암호화해 보내면, A는 자신의 개인키로 복호화해야 한다.

### 71. 일부 평문과 대응 암호문이 가용한 상황의 공격

**정답: ② 알려진 평문 공격**

공격자가 일부 평문과 그에 대응하는 암호문을 알고 있는 상황은 알려진 평문 공격이다. 선택 평문 공격은 공격자가 원하는 평문을 암호화해 볼 수 있는 상황이다.

### 72. `r^e M mod n` 값을 서명자에게 전송하는 RSA 전자서명 기법

**정답: ① 은닉서명**

메시지 `M`에 난수 `r`을 곱해 `r^e M mod n` 형태로 가린 뒤 서명자에게 보내는 구조는 Blind Signature, 즉 은닉서명이다. 서명자는 실제 메시지를 알지 못한 채 서명할 수 있다.

### 73. 위험분석 방법론 연결

**정답: ④ 시나리오법, 순위결정법**

“어떤 사건도 기대대로 발생하지 않을 수 있다”는 전제와 일정 조건을 기반으로 분석하는 방식은 시나리오법이다. 위협요인의 우선순위를 도출하는 방식은 순위결정법으로 정리한다.

### 74. 공개키 암호의 필요성으로 틀린 것은?

**정답: ① 무결성**

공개키 암호가 등장한 주요 필요성은 대칭키의 키 분배와 관리 문제 해결, 인증, 부인방지다. 무결성은 해시나 MAC, 전자서명과 함께 달성되는 보안 목표이지만 공개키 암호의 직접적인 등장 필요성으로 보기에는 상대적으로 거리가 있다.

## 5과목. 정보보안 관리 및 법규

### 85. 정보통신망법 용어 정의

**정답: 이미지 지문 재확인 필요**

해설상 포인트는 다음과 같다.

- 전자문서는 반드시 암호화된 문서를 의미하지 않는다.
- 개인정보는 살아 있는 개인에 관한 정보다.
- 사망자 정보는 일반적으로 개인정보보호법상 개인정보에 해당하지 않는다.

이미지의 ㉠~㉣ 문장을 다시 보고 틀린 조합을 고른다.

### 86. 조직의 정보보호 교육 대상자에 해당되지 않는 사람

**정답: ① 조직의 중요한 고객**

정보보호 교육 대상은 최고경영자, 임직원, 신입직원, 외부 위탁자 또는 조직 정보시스템을 사용하는 외부 이용자 그룹까지 포함될 수 있다. 단순히 “중요한 고객”이라는 이유만으로 조직의 정보보호 교육 대상자로 보기는 어렵다.

### 87. 위험분석 용어 빈칸

**정답: 이미지 지문 재확인 필요**

위험분석에서는 용어를 정확히 구분해야 한다.

- 자산: 보호해야 할 대상
- 위협: 손실을 발생시킬 수 있는 원인
- 취약성: 위협이 악용할 수 있는 약점
- 위험: 위협이 취약성을 이용해 손실을 발생시킬 가능성
- 발생가능성: 위험이 실제로 발생할 확률 또는 빈도

이미지 지문의 문장 구조에 맞춰 빈칸을 다시 연결한다.

### 90. 시스템 보안평가 기준

**정답: ③ CTCPEC**

CTCPEC는 캐나다에서 개발한 보안평가 기준이다. TCSEC는 미국, ITSEC는 유럽, CC는 국제 공통평가기준이다. 문제에서 캐나다 평가 기준과 T0~T7 등급 체계를 말한다면 CTCPEC로 정리한다.

### 95. 관리적 예방대책에 속하는 것은?

**정답: ④ 문서처리 순서의 표준화**

관리적 대책은 정책, 절차, 표준, 교육, 조직 운영 같은 관리 중심 통제다. 안전한 패스워드 강제, 침입차단시스템, VPN은 기술적 대책에 가깝다. 문서처리 순서 표준화는 절차 통제이므로 관리적 예방대책이다.

### 97. 정보보호대책 선정 시 영향을 주는 제약사항

**정답: 이미지 지문 재확인 필요**

정보보호대책 선정에는 법적, 시간적, 환경적, 사회적, 예산상 제약이 모두 영향을 줄 수 있다. 이미지 지문에 “법령·규정 준수”, “기간 내 구현”, “물리적 환경”, “사회적 수용성” 중 어느 단서가 있는지 확인해 연결해야 한다.

### 99. 개인정보 유출사고 통지·신고 의무 설명으로 틀린 것은?

**정답: ② 1천명 이상의 정보주체에 관한 개인정보 유출 시 신고**

정보통신서비스 제공자등은 개인정보 유출 사실을 알게 되면 이용자에게 지체 없이 통지하고, 정당한 사유 없이 24시간을 넘겨서는 안 된다. 해설 기준으로는 정보통신서비스 제공자등의 신고 기준을 1천명 이상으로 제한한 설명이 틀린 보기다.

### 100. 정보보호 거버넌스 국제 표준

**정답: ③ ISO 27014**

ISO 27001은 ISMS 요구사항, ISO 27017/27018은 클라우드 보안 및 개인정보보호와 관련된다. 정보보호 거버넌스 국제 표준은 ISO 27014로 정리한다.
